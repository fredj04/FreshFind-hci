🌿 FreshFind Home Overview Design Process Prototyping Heuristic Eval Reflections

Heuristic Evaluation

Nielsen's 10 Usability Heuristics — findings and design responses

Evaluation Setup

Evaluators

Glenn Miller and Simisola Lawal (peers, CSE 400) evaluated one of the three task flows independently. Friends and family evaluated the remaining flows. Each evaluator reviewed screens independently before findings were consolidated.

Method

Nielsen's 10 Usability Heuristics. Issues recorded on index cards noting: which heuristic was violated and a severity rating. Severity scale: 0 = No problem, 1 = Cosmetic, 2 = Minor, 3 = Major, 4 = Catastrophe.

5 Issues Found

#Heuristic ViolatedIssueSev.
H1 4 — Consistency & Standards Each map pin is color-coded but not shape-differentiated. Colorblind users cannot distinguish between Open, Low Stock, and Closed pins. 0
H2 2 — Match Between System & Real World Non-English-speaking users would have trouble selecting their preferred language if all options are displayed only in English on the language screen. 2
H3 3 — User Control & Freedom Users who post an incorrect status update have no way to delete or correct it before other users see it and act on wrong information. 2
H4 8 — Aesthetic & Minimalist Design The listing detail cards contain too many words, making it harder to quickly find the key information (open status, hours, distance). 1
H5 6 — Recognition Rather Than Recall Users with poor eyesight may find it hard to continuously read text on the screen. Icons alongside text labels would reduce cognitive load and reading effort. 1
Severity scale used:   0 No problem  ·  1 Cosmetic  ·  2 Minor — low priority fix  ·  3 Major — fix before release  ·  4 Catastrophe — must fix before launch

Design Changes Made in Response

C1 H1 · Heuristic 4 — Consistency & Standards · Sev. 1

Map pins were redesigned to use different shapes AND different colors: green circle with a checkmark (Open), orange square with an exclamation mark (Low Stock / Closing Soon), red diamond with an X (Closed). This ensures colorblind users can differentiate pins by shape alone. The floating legend on the map home screen was updated to show both shape and color for each status.

C2 H2 · Heuristic 2 — Match System & Real World · Sev. 3

The language selection screen was updated so that each language is displayed in its own script — "Español" instead of "Spanish," "Kreyòl ayisyen" instead of "Haitian Creole," "中文" instead of "Chinese." A smaller English translation appears below each language name for users who can read both. This ensures that a Spanish speaker who cannot read English can still identify and select their language.

C3 H3 · Heuristic 3 — User Control & Freedom · Sev. 3

An "Undo my update" button was added to the confirmation screen. The button is clearly visible (outlined, not a small gray link) and includes a 5-minute countdown timer showing how long the undo option remains active: "Undo available for 4:56." Once the timer runs out, the button deactivates and disappears. If clicked, the user is returned to the quick-tap status menu to select the correct status. This directly addresses the concern about incorrect updates being seen and believed by other users.

C4 H4 · Heuristic 8 — Aesthetic & Minimalist Design · Sev. 2

Listing detail cards were simplified. The key information — resource name, open/closed status badge, and hours — was moved above the fold. Secondary details (distance, SNAP tag, community timestamp, staff note) appear below. Action buttons (Get Directions, Update Status) were given icons (a navigation arrow and a pencil respectively) to make their function immediately recognizable without reading the label.

C5 H5 · Heuristic 6 — Recognition Rather Than Recall · Sev. 1

Icons were added to all primary action buttons throughout the app. The Get Directions button uses a navigation arrow (➤). The Update Status button uses a pencil icon. Status options in the quick-tap menu each have a paired icon: a checkmark for Open, a clock for Closing Soon, an X for Closed, and a warning triangle for Out of Fresh Produce. This reduces reading load for users with low literacy or poor eyesight.
